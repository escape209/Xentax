long    boneIndex

Code: Select all

File: g_uec.ve1
Branch 0
Bone Count: 45
Sibling Layouts:
Address: 0xC

float    Rotation[4] // Quaternion
float    Translation[3]
float    Scaling[3]
byte     parentIndex
byte     skip[3]

Branch 1
Bone Count: 5
Sibling Layouts:
Address: 0x7C8

float    Rotation[4] // Quaternion
float    Translation[3]
byte     parentIndex
byte     skip[3]

// BigEndian
// Quaternion
// Local Space
// Column-major
// Meter
// Y, Z, X
This is a case where the concept of Branch is involved.



Since the scaling component is omitted in the second Branch, we can't handle it with a single layout while preserving all the explicit attributes, but there's a workaround if we just skip the scaling component.

Code: Select all

// workaround B
Bone Count: 50
Sibling Layouts:
Address: 0xC

float    Rotation[4] // Quaternion
float    Translation[3]
test     length[<][45]
long     skip[3]
calc     length[+][1]
byte     parentIndex
byte     skip[3]
