Post by Mr.Mouse » Thu Oct 12, 2006 12:06 pm

TsunamiZ wrote:whats the front and back?
A file is a very long string of bytes.

For instance, a file could look like this, each letter represents one byte.

Code: Select all

SDFHFNGNBCVCXZXCVVNM12837485E
The first byte, S, is at position 1 (or 0 for geeks).

So:

Code: Select all

SDFHFNGNBCVCXZXCVVNM12837485E
123456789....and so on.....
FRONT.......MIDDLE.......BACK
The second byte is D, see?

So the FRONT of the file starts at the S byte.
The end of the file starts at byte E (the last byte in the string).

If we say we need the front and the back of a file we need a number of bytes from the start (FRONT) of the file (in example say "SDFHFN"), na from the end (BACK) of the file (in example say "37485E").

The file cutter tool can cut of pieces from the start and from the end, it will zip it and you can upload that zip file.

Why can we use only that and not the whole file? Because important information about the contents of archives is usually saved at the front (start) or at the back (end) of the archive.
