Post by aluigi » Thu Mar 10, 2011 5:24 pm

problem found.
the explanation is located in the encryption algorithm that is a block cipher.

in short a block cipher doesn't work on single bytes but it works on sequences of bytes like 8 bytes each time.

now if the original compressed size of the file in the archive was 0x123 then the game will decrypt 0x120 bytes and will decompress 0x123 bytes.

if our file was compressed better than the original like 0x11f bytes then the game will do the same as before, so decrypting 0x120 bytes and decompressing 0x123 (because we have only reimported the file, not changed its zsize/size values).

obviously this leads to the corruption of the final bytes of the compressed data because it should decrypt 0x118 bytes and not 0x120 otherwise the remaining 7 bytes that are in plain-text will be decrypted and so corrupted.

well, all this technical explanation is the cause of the problem, hope at least someone has appreciated it :)

anyway don't worry the fix in quickbms occupies only some lines so I will release it in the next hours in case I remember something else to change.

thanx for the feedback, through this crysis 2 I'm making the new reimporting function of quickbms really error-proof ih ih ih :)
