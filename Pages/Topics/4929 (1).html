moonlordx wrote:still damage bug after inserting string back
I think its kinda stupid if is anti-cheat feature, coz editing language file
has nothing to do with cheating
they using maybe some crc checksum protection for checking if file is original:-/
The game use CRC32 to check every file. So if you modify some file, you must also modify the CRC32 Checksum in the exe file.
Here is how to do it in Noob way:
Origin Checksum: use winrar to compress the origin file you want to modify. Then in "CRC32" column, you'll see the Checksum. copy it down.
Use the same way to get the checksum for your new file.

Now use Hex edit tool(hexshop,ultraedit, anything you like)open wormreloaded.exe, search for hex value of Origin Checksum. Remember,the byte order should be reversed. If the checksum is 0x12345678, then search for 78 56 34 12.
Why? Go wiki for"endian".
Usually you'll find this four bytes only once. If so, it's where the checksum is. Recored the address. If it happens two or three times, just try to mod the value.Then put your checksum in this place.Remember to reverse it. Also, the address would be invalid after each update. you'll need to do it all over again for new update.

The game uses crc32 checking. So here's a tricky solution:

1: Choose you file, for example: AllTextEng.bin
1: Make a backup copy of the original file (for example: AllTextEng.bin.original)
2: Translate the file as you wish (for example with the caws tool)
3: Correct the translated file with my program, using the original file as a reference value,
for example: crc32trick.exe AllTextEng.bin AllTextEng.bin.original
(This is a commandline program, no graphical interface.) http://www.sendspace.com/file/e624nh
