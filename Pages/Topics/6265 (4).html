Post by aluigi » Sun Apr 10, 2011 4:18 pm

-r -w are options so must be placed outside the " quotes and then you have added tons of paths... why?
the format is simply: quickbms.exe options script.bms archive folder

anyway the following is a translation from the italian of the step-by-step I gave to an user on my forum:
  • in the folder where you have extracted the files delete those that you have not modified, for example if you have modified the file my_folder\settings\settings.xml then delete all the others and leave only this one (don't touch its path)
  • create a backup copy of the pak file you will modify
  • create a link to quickbms.exe and add -r -w in the "Target:" field in the properties of the link, for example the result should be like: "c:\folder\quickbms.exe" -r -w
  • click on the link you have just created
  • quickbms will guide you step-by-step, so select the crysis2.bms file
  • select the original archive in which you want to reimport the files you have modified
  • select the folder where the files reside, exactly the same folder you selected for the extraction
  • at the end of the process quickbms will tell you how many files have been reimported, so verify that they are exactly those you expected
as you can see it's really a joke to use the reimport feature, indeed except for the link it's EXACTLY the same of the extraction

Post by aluigi » Thu Apr 21, 2011 12:44 pm

the files are compressed and with some of them the compression used by Crysis is better than the one used by quickbms (zlib used in 7zip) while for others is better the one of quickbms.

but in this case I guess it's just the problem of the different chars used for the translation that occupy more space during the compression: english takes the chars under 0x7f while brazilian uses also accents and other chars making the compressed result bigger.

so for the compressed files it's important that both the real and compressed sizes are lower/equale than the originals.
