Post by gildor » Mon May 16, 2011 2:58 pm

Szkaradek123 wrote: Most of games based on Unreal Engine after unpacking using Gildor's umodel, create folder with materials ,
which are reading by script .
Noesis - do not. So images must be manual imported.
PKS format has support for mapping a texture to surface. There is no way to convert UE3 material to a single texture - it has diffuse, bump, opacity etc. ".mat file" is my own extension which allows me to load relatively complex materials similar to UE3 into 3ds Max automatically. For PSK files produced by other tools (not umodel) you usually have to create and assign materials by hands.
almondega wrote: Your script is already faster than Gildors one for 3DS max, congratz.
They are incomparable Blender uses Python for scripting, which should be faster than MaxScript. The slowest thing in PSK mesh loading into Max is assignment of weights to vertices, which is done by simulating GUI clicks (ugly way, it looks like MaxScript API were designed by "strange people"). I hope Blender were designed in a better way.
