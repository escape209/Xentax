HI first post here, so ive extracted everything, sadly anything from the .pak68 archives cant be opened properly in blender as far as I know as noesis doesnt like the ASCII files it generates, also once inside the .pak you want the files arent named properly, theyre just called sea-arrival-river-vegetation_1 etc etc


for example lets say you want to extract some lovely ferns from the seattle intro bit, you find the .pak you want inside the .pak68 folder and open the text file associated with it "sea-arrival-river-vegetation.txt" this file contains the contents of that .pak

the contents of that file reads as follows:

level-id sea-arrival-river-vegetation
package sea-arrival-river-vegetation
package sea-arrival-river-vegetation-ingame
package sea-arrival-river-vegetation-phys
package sea-arrival-river-vegetation-dict
cache sea-arrival-river
cache sea-arrival-river-vegetation-trees
cache sea-arrival-river-vegetation-forest
cache sea-arrival-freeway-vegetation-trees
actor bush-shrub-a-sapling-a
actor bush-shrub-a-sapling-b
actor bush-shrub-a-sprig-a
actor bush-shrub-a-sprig-b
actor bush-shrub-a-sprig-c
actor plant-fern-bracken-b
actor plant-fern-bracken-crouch-a
actor plant-fern-bracken-prone-a
actor shrub-fern-a-2m-crouch-cover
actor shrub-fern-a-a
actor shrub-fern-a-a-sm
actor shrub-fern-a-b
actor shrub-fern-a-c

Each level contains its respected assets and the .txt tells you what assets are inside the level, but.... when you extract the respective .pak from the pak68 folder so in this case youd go to \Image0\build\ps4\main\world-seattle-arrival_files\pak68\sea-arrival-river.pak and extract it via the tlou2.exe tool, it contains ASCII and smd but NONE OF THE ASCIIS ARE USABLE, they cannot be opened or previewed inside noesis even with the xps script, the SMD's contain only 1 UV map and most of the models have multiple sets for things like grunges and masks etc, the ASCIIS contain multiple UV maps and are thus needed over the smd files,

Can anyone help? I'm having a nightmare here trying to get this work, so far the only workaround ive found is looking for the asset I want inside the actor97 folder for each respective level, search for the ACTUAL asset name, not what its called inside its subfolder.

for example, if I want a fern, i open the .txt file for the level: "sea-arrival-river-vegetation.txt" then look for the asset I want, in this case its: "actor plant-fern-bracken-crouch-a" make sure you search for your assets INSIDE the actor97 folders or they wont work.

note: ASCII seems to extract and work *fine* even with multiple UV's if extracted from the actor97 folders, but for some reason, anything ASCII from the pak68 folders is broken with this tool. I've managed to get the sedan working and even set up its textures in blender if anyones interested.






NOTE: notice how the model has multiple UV sets BECAUSE it was imported as ASCII and extracted from the actor97 folder. You cannot do this AS I KNOW OF with anything from the pak.68 folders.
