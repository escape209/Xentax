normally the textures are right after the model file.
i output the quickbms log to a txt file and use that to find the textures.
use the command line to extract it and add
> c:\mylog.txt
and it will log the output to that txt file.

einherjar007 wrote: ↑Sun Jul 18, 2021 3:38 am looks like a zlib(78 DA) files.
Also, BILZ (42494C5A) is look familiar. Isn't the file compressed?
I'm sure someone had already written the decompression script.

As an aside, this game uses havok for animation.
I saw hka before, but it uses a rare compression("PREDICTIVE_COMPRESSED") and the plugin doesn't support it.
The zlib (BLIZ) files from the arcade dump can be decompressed with this scrip from Aluigi:

https://aluigi.altervista.org/bms/gundam_breaker_3.bms

After that you can use chroxx plugin and noesis to view the models(in this example the file pl01_01_all.mdl was extracted):
GS3.jpg (182.59 KiB) Viewed 329 times
All of them would need to be extracted manually and renamed accordingly but at least it works
