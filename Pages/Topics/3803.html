Post by GameZelda » Sat Oct 24, 2009 11:26 pm

eng_us.loc looks like a very simple string table format.

First there's a LOCH section:

Code: Select all

struct LOCH
{
   char magic[4]; // "LOCH"
   uint32_t size;
   uint32_t unk1; // 0
   uint32_t unk2; // 1.
   uint32_t unk3; // 20.
}
Then a LOCL section:

Code: Select all

struct LOCL
{
   char magic[4]; // "LOCL"
   uint32_t size;
   uint32_t unk1; // 0
   uint32_t nStrings;
   uint32_t stringPtr[nStrings]; // Pointers to little endian unicode strings, relative to the start of the LOCL section
}
