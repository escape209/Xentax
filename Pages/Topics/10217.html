Post by Axsis » Sat Mar 30, 2013 1:24 pm

Mul files are very similar to Deus Ex 3 muls - every mul contain several fsb files (1-6) in which there are mono mp3 files (in PC version) - but with new starting offset to data (0x2000), so old tools doesn't work.
To extract one stereo track you should do this conversions - mul -> 2x fsb -> 2x mp3(mono) -> mp3(stereo).
Another problem is that music in TR is dynamic - there are 730+ files in "pc-w\audio\streams\music\" so manual conversion will be long & painful...
Next pain is that in fsb files there are names of mp3 samples, but these names are truncated to 29 chars. Sometimes when you extract left and right mp3s from two fsbs using fsb_mpeg it will overwrite one mp3 with another due to identical truncated names. To prevent this, you should move fsb files in different dirs or rename first mp3 (left) before extracting second (right).
Anyways, if anyone still want to convert some muls to mp3s here is the script:

Code: Select all

# MUL demuxing script v0.2 by Axsis
# Format description from Gibbed.DeusEx3.Demux/Program.cs by Rick
Endian little
Set EXT string "fsb"
Set START_OFFS long 0x2000

Get FSIZE asize
Get NAME basename
GoTo 0x0C
Get STREAMS long
PutVarChr MEMORY_FILE FSIZE 0
For S = 0 < STREAMS
 Print "Unpacking stream %S%/%STREAMS%"
 Log MEMORY_FILE 0 0
 Math STREAM_SIZE = 0
 Math OFFS = START_OFFS
 Do
  GoTo OFFS
  Get SEGMENT_TYPE long
  Get SEGMENT_SIZE long
  GetDString DUMMY 8
  #Print "STYPE:%SEGMENT_TYPE%;   SSIZE:%SEGMENT_SIZE%"
  If SEGMENT_TYPE == 0
   For B = 0 < STREAMS
    Get BLOCK_SIZE long
    Get BLOCK_STREAM long
    GetDString DUMMY 8
    #Print "\tBSIZE:%BLOCK_SIZE%;   BSTRM:%BLOCK_STREAM%"
    If BLOCK_STREAM == S
     SavePos OFFS
     Append
     Log MEMORY_FILE OFFS BLOCK_SIZE
     Append
     Math STREAM_SIZE += BLOCK_SIZE
    EndIf
    GoTo BLOCK_SIZE 0 SEEK_CUR
   Next B
  Else
   #Print "STYPE:%SEGMENT_TYPE%;   SSIZE:%SEGMENT_SIZE%"
   GoTo SEGMENT_SIZE 0 SEEK_CUR
  EndIf
  SavePos OFFS
  Math OFFS x= 0x10
 While OFFS < FSIZE
 String FNAME p= "%s_stream%u.%s" NAME S EXT
 Log FNAME 0 STREAM_SIZE MEMORY_FILE
Next S
fsb_mpeg is here: http://hcs64.com/vgm_ripping.html (but i think you already have it, if you're reading this topic ) run it with -p 16 to set padding (fsb_mpeg.exe blablabla.fsb -p 16)

PS: Probably will work with X360 muls - just change first two lines to "Endian big" and "Set EXT string "xma"" but I didn't tested.

PS2: Should also work with DeusEx3 if change "Set START_OFFS long 0x2000" to "Set START_OFFS long 0x800"
