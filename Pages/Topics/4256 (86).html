yoda wrote:nirgal : i know that all the lods can be extracted ,my question is about import all the lods extracted with the last released made by Ernegien, or still only the lod 0
Forza studio does not support the importing of any model in-game, if that's what you're talking about.
sitzuhg wrote:I still have problem with digitalGauge-textures. There are no UVs as it seams. All other textures are working fine. But Speedo-textures and so on do not work. Tried in multiple cars. Also tried with autovista models. Anyone else has this problem?

It should make no difference to the game what the actual size of the wheels are, as it just calculates the bounding box dimensions of the current model and scales appropriately for the desired wheel size. The same is most likely true for calipers and rotors, which will need to vary based on the wheel size. I'm assuming all of this information would be stored in their SQLite database file per vehicle.

Model lod0 = nodamage_lod0.xds
Cockpit Lod0 = interior_lod0.xds , gauges uses interior_lod0.xds
Lights lod0 = lights_lod0.xds
Lod 1/2/3/4/5 = nodamage.xds - interior.xds - lights.xds
Wheel Lod0 = wheel_lod0.xds
Wheel Lod1/2/3/4/5 wheel.xds
Tire only on lod1/2/3/4/5 but in some cars is deformed the tire and the tires for wheels upgrade the same
