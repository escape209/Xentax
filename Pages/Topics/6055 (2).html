Now for the Idea,
as I was repacking the "nigel" file from the demo,
so often, I recognised that when you reimport
files it takes pretty long...

Normal behavior:
QBMS looks inside the file you want to import
the data in. Then it searches for all files
and dirs wich are in the file and then
it compares the files on by one with
the files you want to reimport.
If the file/dir of the e.g. nigel
is the same like the file you want to reimport
then it reimports it. Then it goes over to
the next file an so on....
For example you reimport *nothing*,
QBMS goes threw all dirs/subdirs/files
wich are in the nigel and compares them with the empty folder.
This takes ~30-50sec, for doing nothing.
I dont want to know how long this would take if your
file would be more then a few gigabytes...

So, wouldn´t it be smarter to switch the procedure ?

Better behavior (probably xD):
QBMS first looks inside the folder
with the files I want to reimport,
it remembers the files/dirs.
After that it looks inside e.g.
the "nigel" and compares
the remembered files/dirs
with the files in the nigel.
After that it reimports the files
that match to the nigel´s files.

This way it would require a few secs only...
