I used this post as a basis to write a batch script to copy the skeletons into the models. It's slow, it's a mess, and don't run it more than once on the same files, but it'll save some manual labor and let you quickly look through your extracted data with noesis to find what you want. Someone else who knows better about writing batch scripts can tidy it up if they want.

Get and run Blader's batch script first. This is mandatory!

It works by creating a size-ordered list of files to determine which is the 'main' model file. Then it looks for a file with "_skel.ascii" in the name and combines the two. Tested and works locally.

-Place this in the same folder as Detroit.exe and run it after you already extracted some data and sorted it into folders with the other batch script
-Uses the same files.txt as Blader's script
-The combined file will be called like "(filename).ascii_withskel.ascii" because I'm not smart

LOUDER FOR THOSE IN THE BACK: USE THE OTHER BATCH SCRIPT FIRST, AND DON'T RUN THIS MORE THAN ONCE ON THE SAME FILES

Code: Select all

@echo off
setlocal EnableDelayedExpansion
set mypath=%cd%
for /f "tokens=*" %%A in (files.txt) do call :process %%A
del /f /q {temp}
exit /b
:process
SET _biggest=0
SET _skel=0
SET _modelname=%~1
IF EXIST "%mypath%\%_modelname%" (
echo processing !_modelname!
dir /b "%mypath%\%_modelname%" /o-s>{temp}
SET /p _biggest=<"{temp}"
FOR /f "tokens=*" %%F in ('find /i "_skel.ascii" {temp}') do (SET _skel=%%F)
IF EXIST "%mypath%\%_modelname%\!_skel!" (
(
FOR /f "tokens=* skip=1 usebackq" %%B in ("%mypath%\%_modelname%\!_biggest!") do @echo %%B
)>"%mypath%\%_modelname%\temp2.file"
copy "%mypath%\%_modelname%\!_skel!"+"%mypath%\%_modelname%\temp2.file" "%mypath%\!_modelname!\temp.file"
ren "%mypath%\!_modelname!\temp.file" "!_biggest!_withskel.ascii"
del /f /q "%mypath%\!_modelname!\temp2.file"
)
)

For models that don't load due to bad .ascii format (i.e. the trees in Hank's house map), here is a python script that fixes them.

Code: Select all

import os
sourceFile = open("FILENAME.ascii", "r")
filename = os.path.splitext(sourceFile.name)[0]
newfilename = "%s_fixed.txt" % filename
with open("%s" % newfilename, "w") as f:
  for x in sourceFile:
    line = x.split()
    if len(line) < 5:
      f.write(x)
  f.close()
The problem is that the .ascii files contain additional unexpected lines. By deleting these lines, the problematic trees can load, and so the map as a whole can load.

detroit_img.exe sometimes randomly fails to convert a texture without throwing any error message. And i don't mean that if fails on some specific files, no, you can run the tool on the same file multiple times in a row, and sometimes it gets converted, and other times it doesn't.

Here's a script that will attempt to convert each texture file (in specified folder and all it's subfolders) until it is successfully converted or the maximum number of attempts is reached. Just change these paths and run:
extractedDir = "D:\\Games\\Detroit\\tools\\extracted"
converter = "D:\\Games\\Detroit\\tools\\detroit_img.exe"

Code: Select all

import os
import subprocess

extractedDir = "D:\\Games\\Detroit\\tools\\extracted"
converter = "D:\\Games\\Detroit\\tools\\detroit_img.exe"
extractAttempts = 20

def convertTextures(dirPath):
    dList = [ d for d in os.listdir(dirPath) if ( os.path.isdir( os.path.join(dirPath, d) ) ) ]
    for d in dList:
        subD = os.path.join(dirPath, d)
        convertTextures(subD)
    for f in os.listdir(dirPath):
        fPath = os.path.join(dirPath, f)
        if not ( os.path.isdir(fPath) ):
            if ( len( f.split(".") ) == 1 ):
                print( "Converting: {}".format(fPath) )
                counter = 0
                while True:
                    if ( counter > extractAttempts ):
                        print( "!!!\nUnable to convert file: {}\n!!!".format(fPath) )
                        break
                    subprocess.run([converter, fPath])
                    counter += 1
                    if ( os.path.exists(fPath + ".tga") ): break

convertTextures(extractedDir)
