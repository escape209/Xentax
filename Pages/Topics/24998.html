I've been going over all the post I've found here and on Zenhax, and used both the new killzone shadowfall and liberation BMS scripts.
I keep coming back to the same error of having the wrong offset output.
It seems to be an error with the Math function or
By chance does anyone know a fix or have the corrected code?
(I am new to Hex with Quickbms, and have not looked at the hex code on the files.)



Shadowfall code used:

Code: Select all

comtype lz4
append

get NAME basename
string name + ".unp"

  get magic long
  get blocksize long 
  get fullusize longlong

  get hash longlong
  get hash longlong

  set ni = fullusize
  math ni / blocksize
  math ni + 1

  for i = 0 < ni
    get PSIZE long
    putarray 0 i psize
  next i

  for i = 0 < ni
    getarray psize 0 i

    savepos OFFSET
    if fullusize > blocksize
      clog name OFFSET PSIZE blocksize
    else
      clog name OFFSET PSIZE fullusize
    endif

    math offset + psize
    goto offset
    
    math fullusize - blocksize
 
  next i
Liberation code used:

Code: Select all

# Killzone Liberation (script 0.2.2)
# script for QuickBMS http://quickbms.aluigi.org

get ARCHIVE_NAME basename

get INFO_SIZE long
get DUMMY long
get DUMMY long
get DUMMY long
get ZERO long
get ZERO long

get NAMES_SIZE long
savepos OFFSET
log MEMORY_FILE OFFSET NAMES_SIZE

math OFFSET + NAMES_SIZE
goto OFFSET

get INFO_SIZE long
savepos OFFSET
log MEMORY_FILE2 OFFSET INFO_SIZE

get ENTRIES long MEMORY_FILE2
savepos OFFSET MEMORY_FILE2
set PATH string ""
set NAME string ""
putarray 0 0 0  # nested function
callfunction EXTRACT

startfunction EXTRACT
    savepos BCK_OFF MEMORY_FILE2
    goto OFFSET MEMORY_FILE2
    string PATH + NAME
    string PATH + /
    for CUR_ENTRIES = ENTRIES > 0
        get NAME_OFF long MEMORY_FILE2
        math FLAGS = NAME_OFF
        xmath NAME_OFF "NAME_OFF & ((1 << 23) - 1)"
        goto NAME_OFF MEMORY_FILE
        get NAME string MEMORY_FILE
        get OFFSET long MEMORY_FILE2
        if FLAGS & 0x80000000
            get ENTRIES long MEMORY_FILE2
            callfunction EXTRACT
        else
            callfunction HANDLE_PACK_NUM 1
            get SIZE long MEMORY_FILE2
            string NAME p "%s%s" PATH NAME
            set EXT extension NAME
            if EXT == "core" && ARCHIVE_NAME == "psp"
                goto OFFSET
                get ZERO long   # probably XSIZE64
                get XSIZE long
                reverselong XSIZE
                math OFFSET + 8
                math SIZE   - 8
                clog NAME OFFSET SIZE XSIZE
            else
                log NAME OFFSET SIZE
            endif
        endif
    prev CUR_ENTRIES
    goto BCK_OFF MEMORY_FILE2
endfunction

startfunction HANDLE_PACK_NUM
    xmath PACK_NUM "(FLAGS >> 23) & 0x1f"
    getarray LAST_PACK_NUM 0 0
    if PACK_NUM != LAST_PACK_NUM
        putarray 0 0 PACK_NUM
        get EXT extension
        if PACK_NUM == 0
            string TMP p "%s.%s" ARCHIVE_NAME EXT
        else
            string TMP p "%s%02d.%s" ARCHIVE_NAME PACK_NUM EXT
        endif
        open FDSE TMP
    endif
endfunction

Post by n1nonly1 » Sun Feb 20, 2022 8:20 pm

after many weeks, i was finally able to figure out a way to extract the model.

the problem is you only the faces and vertices, but no uvs (renderdoc captures based on perspective)

Requirements/Programs needed:

-renderdoc(complete installation)
-rpcs3
-killzone 2
blender 2.8-2.92 (2.92 strongly recommended)
-custom blender import_pix addon (https://github.com/sbobovyc/GameTools/b ... ort_pix.py)


-get renderdoc to attach to rpcs3.exe
-launch rpcs3, then killzone 2 (vulkan api)
-head into the level to where your model is
-once close to the model (to the point where its rendered), hit f12
-after renderdoc captures, you can close rpcs3

-use renderdoc timeline to locate your model (mesh viewer)
-once located, export to .csv all files of the model
-close renderdoc

-open blender (must be versions 2.8 - 2.92, else wont work because of deprecated functions in newer versions)
-install the addon

-finally, import the .csv files using the addons.
-if the model is large, there may be more than 1 file to import (as it was in my case)

for more info, please dm me
