There is nothing special in structure:

Header:
4B - Version
2B - Unknown
4B - ItemCount

Items (array[ItemsCount]):
4B - ItemID
4B - String Length
4B - Block Length (it is real block size used for Unicode/utf-8 bytes)
4B - Offset
4B - Unkown

String block starts and ends with separator value 0x0A0B0C0D

regarding the ogg music files, i have successfully converted them into properly playable OGGs. Here's how:
  1. Download the archive extractor that RaptorX posted on page one: viewtopic.php?f=10&t=6743#p55150
  2. Run DungeonSiege3Extractor.exe and use it to open data_archive.oaf in the DS3 root
  3. Show only OGG files using the file type dropdown, CTRL-A to select all, save selected to somewhere. Note that the files are extracted into their existing directory structure, i.e. \global\audio\sound_bank
  4. Download the program ww2ogg from http://hcs64.com/vgm_ripping.html and put it in the directory with your extracted OGGs. You only need the files ww2ogg.exe and packed_codebooks.bin from this archive.
  5. Download revorb from http://www.hydrogenaudio.org/forums/lof ... 64328.html and put it in the directory with all the OGGs. This program fixes the "granulepos", which is apparently used by some players to determine duration.
  6. Save my attached batch in the directory with all the OGGs and then run it.
  7. Step 3: Profit!
